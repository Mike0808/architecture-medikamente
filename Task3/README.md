# Задание 3. Оценка Data Encryption at Rest and In Transit

## Классификация и защита


| Категория данных                                                                     | Почему необходимо защищать                                                                                                       | Тип защиты                                                                                               | Инструменты и методы                                                      |
| ------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| Персональные данные (PII)<br>ФИО, телефон, e-mail, адрес, паспорт, дата рождения | Закон ФЗ-152, риск утечки ПДн и репутационных потерь; злоумышленники могут использовать для социальной инженерии и мошенничества | At Rest: колоночное и файловое шифрование;<br>In Transit: TLS 1.2 / mTLS                         | pgcrypto / Postgres TDE; KMS Vault/HSM; TLS 1.2; JWT + OIDC для доступа |
| Медицинские данные (PHI)<br>Диагнозы, назначения, результаты анализов            | Закон ФЗ-323; это самая чувствительная категория данных; раскрытие наносит прямой ущерб пациенту                                 | At Rest: шифрование файлов в S3 (SSE-KMS), шифрование в EHR БД;<br>In Transit: mTLS внутри сети  | S3/MinIO SSE-KMS; Postgres TDE; TLS 1.2 + mTLS; Kafka SASL_SSL            |
| Финансовые данные<br>Платежи, реквизиты, чеки, ИНН                               | Требования ФЗ-115 (ПОД/ФТ) и банковских стандартов; утечка → мошенничество                                                       | At Rest: шифрование в 1С;<br>In Transit: TLS между Billing <-> Adapter-1C <->  ККМ       | TLS 1.2; mTLS; Vault для ключей; PGP для архивов; логирование в WORM      |
| Документы и сканы<br>PDF, JPG, DICOM-файлы                                       | Часто содержат ПДн + PHI в одном файле; утечка → компромат на пациента                                                           | At Rest: шифрование объектов в S3 через SSE-KMS, Object Lock;<br>In Transit: presigned URL + TLS | MinIO SSE-KMS, AV/DLP, presigned URL TTL≤600 s, HTTPS TLS 1.2             |
| Операционные данные<br>Логи, метаданные, lineage                                 | Могут содержать токены и PII в метаданных; важно сохранить целостность и невозможность подмены                                   | At Rest: WORM-хранилище, цифровая подпись;<br>In Transit: TLS 1.2                                | Loki/VM + S3 WORM, pgp-подписи, TLS 1.2                                   |
| Технические данные<br>Конфиги, секреты, ключи API                                | Компрометация даёт полный доступ к инфраструктуре                                                                                | At Rest: хранение в Vault/HSM;<br>In Transit: TLS + ротация ключей                               | HashiCorp Vault, KMIP, HSM, TLS 1.2 + OCSP Stapling                       |
